Final Results Table

What parts worked out as expected?

Thanks to our prudent planning most of this project worked out as planned. The inputs and outputs all worked as intended, we had no issues there. Implementing the Package class worked well, it also helped cut down on unit test code. The unit tests worked as intended, by helping us identify small logical errors that would have went uncaught otherwise. The Python interpreter can only catch syntax errors and fatal logic errors.

What parts did not work out as expected?

Like with all programming projects we faced problems. The first problem we faced was extremely high quotes. Our program was outputting quotes of €5000 for a car worth only €4000. This was caused by compounding, every time we increased the quote we multiplied the same variable causing the value to compound. We fixed this by implementing a rate_accumulator array. Each quote increase is appended to this list and at the end the get_quote function returns the sum of the quote_accumulator, using the Python sum() function.

What were our biggest challenges?

Our biggest challenge was using the csv to store our unit test values. We could not get this to work properly for two classes as we were trying to use our own functions. After further research we found that Python has a built in csv module to help deal with these problems, thus allowing us to cut down on code length and complexity.
