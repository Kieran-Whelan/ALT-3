Design

Problem Breakdown

We used decomposition and abstraction to design this model. We initially ignored the fine details and optimizations to create a prototype version. We broke down the problem into the following requirements:

  • Step 1 - We need variables to base the quote off

  • Step 2 - We need to take in these variables as input from the user

  • Step 3 - We need to pass these variables into a function that calculates and returns the quote

  • Step 4 - We then need to display the quote to the user

Code Design

We will use Python 3 to code the model. We chose snake_case as our variable and function naming convention throughout this project and Pascal case for out class naming convention. We will employ a combination of functional and object orientated programming, we decided to use an object oriented approach as it will br practical for implementing unit tests. We will use the Python built-in "math" and "csv" libraries, thus cutting down on code length.
